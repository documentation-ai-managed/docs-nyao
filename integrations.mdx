## Git and Version Control Integration

Connect Nyao to GitHub, GitLab, or Bitbucket to automatically sync documentation changes with your repository.

<Steps>
  <Step title="Connect Repository" icon="git-branch">
    Navigate to your project settings and select `Integrations > Git`.
    
    Enter your repository URL, such as `https://github.com/your-org/your-repo.git`.
  </Step>
  <Step title="Authenticate" icon="lock">
    Choose OAuth or personal access token authentication.
    
    <CodeGroup tabs="GitHub,GitLab">
    ````bash
    # GitHub Personal Access Token
    export GITHUB_TOKEN=your-token-here
    ````
    ````bash
    # GitLab Access Token
    export GITLAB_TOKEN=your-token-here
    ````
    </CodeGroup>
  </Step>
  <Step title="Sync Changes" icon="git-commit">
    Enable auto-sync. Nyao pulls updates on every push to your main branch.
  </Step>
</Steps>

<Callout kind="tip">
  Use branch protection rules in your Git provider to ensure documentation updates follow your team's workflow.
</Callout>

## API Endpoints for Custom Extensions

Nyao provides REST API endpoints to create custom integrations. All requests use HTTPS and require authentication.

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token: `Bearer YOUR_API_KEY`
</ParamField>

<ParamField path="projects/{projectId}/docs" param-type="string" required="true">
  Project ID from your Nyao dashboard.
</ParamField>

Here's a sample request to fetch documents:

<CodeGroup tabs="JavaScript,Python">
````javascript
const response = await fetch('https://api.example.com/v1/projects/{projectId}/docs', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  }
});
const docs = await response.json();
console.log(docs);
````
````python
import requests

headers = {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
}
response = requests.get('https://api.example.com/v1/projects/{projectId}/docs', headers=headers)
docs = response.json()
print(docs)
````
</CodeGroup>

## Webhook Setup for Notifications

Set up webhooks to trigger actions on events like document updates or page views.

### Basic Setup

<Steps>
  <Step title="Create Webhook" icon="zap">
    Go to `Project Settings > Webhooks > New Webhook`.
    
    Enter your endpoint URL: `https://your-webhook-url.com/nyao`.
  </Step>
  <Step title="Select Events" icon="bell">
    Choose events such as `doc.updated` or `page.viewed`.
  </Step>
  <Step title="Test and Save" icon="play">
    Send a test payload and verify receipt.
  </Step>
</Steps>

Sample webhook payload:

````json
{
  "event": "doc.updated",
  "projectId": "proj_123",
  "docId": "doc_456",
  "timestamp": "2024-01-15T10:30:00Z"
}
````

<Expandable title="Advanced Webhook Configuration" default-open="false">
  Add secret validation:
  
  <ParamField header="X-Nyao-Signature" param-type="string" required="false">
    HMAC SHA-256 signature for payload verification.
  </ParamField>
</Expandable>

## Third-Party App Connections

Integrate Nyao with popular tools using pre-built connectors.

<Tabs>
  <Tab title="Slack" icon="message-circle">
    Post document updates to a Slack channel.
    
    <Steps>
      <Step title="Add to Slack">
        Use `/nyao connect` in your workspace or visit the Nyao Slack app page.
      </Step>
      <Step title="Select Channel">
        Choose where notifications appear.
      </Step>
    </Steps>
    
    Example payload:
    ````json
    {
      "text": "New doc published: *Getting Started*"
    }
    ````
  </Tab>
  <Tab title="Jira" icon="package">
    Link docs to Jira issues for better tracking.
    
    <CodeGroup tabs="Create Issue,Link Doc">
    ````javascript
    // Create Jira issue from Nyao doc
    await fetch('https://your-jira.atlassian.net/rest/api/3/issue', {
      method: 'POST',
      headers: { 'Authorization': 'Basic ' + btoa('email:api-token') },
      body: JSON.stringify({
        fields: { summary: 'Review Nyao doc', project: { key: 'DOCS' } }
      })
    });
    ````
    ````javascript
    // Add doc link to existing issue
    await fetch('https://your-jira.atlassian.net/rest/api/3/issue/ISSUE-123', {
      method: 'PUT',
      body: JSON.stringify({
        update: {
          description: [{ set: 'Nyao doc: https://docs.example.com/guide' }]
        }
      })
    });
    ````
    </CodeGroup>
  </Tab>
</Tabs>

<Callout kind="alert">
  Always store API keys securely and rotate them regularly. Use environment variables in production.
</Callout>
